# 5. 단계별 앱 기획 및 개발 고도화 전략

모든 앱 기획 및 개발은 단번에 모든 기능을 개발하는 것이 아니라 단계별로 사용자의 피드백을 받으며 고도화 해나가는 플랜으로 개발하도록 기획해줘
앱 기획은 항상 최대한 간단하게 개발해볼 수 있는 형태로 기획을 하고 향후 단계별 로드맵 계획을 세워서 고도화 해나갈 수 있게 기획해줘
시작이 너무 과하면 앱 출시도 못해보고 망할 수 있기 때문에 신중하게 간단하게 개발하고 유저 반응보고 고도화해 나가는 전략을 추구하고 싶어
가능하면 게임 앱이 아닌 일반 앱은 핫한 AI 앱 위주로 개발하고 싶어
앱 배포 CI/CD 는 자동화 환경을 구축해서 개발과 배포가 아주 간편할 수 있도록 개발/배포/운영 아키텍처를 구성하도록 규칙을 추가해줘

모든 앱은 토스 앱인토스로 개발할 예정이야 토스 앱인토스를 사용하는 방향으로만 기획해줘

이 규칙을 반드시 기억해줘

# 25. 상세 기획 코드 간소화 제안

각 프로젝트별 상세 기획 템플릿에 코드까지 들어가니까 문서가 너무 복잡해 어떤 기술 스택으로 어떻게 구현하겠다는 내용과 FLOW 정도만 있고 코드들은 제거하는게 좋을거같아
코드는 실제 프로젝트를 진행하면서 실제 프로젝트에서 작성 하는게 좋을거 같아 상세 기획서가 코드 때문에 불필요하게 너무 복잡해지는거 같아
실제 기획 내용이 정확하게 전달될 수 있도록 코드는 제거하는게 좋을 거 같아
상세 기획 템플릿에서 코드를 제거하고 코드 부분을 간소하게 표현할 수 있게 개선 방안을 제안해줘
