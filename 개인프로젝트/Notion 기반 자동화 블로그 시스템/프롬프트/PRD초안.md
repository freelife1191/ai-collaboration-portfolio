### **Notion 기반 자동화 블로그 개발 PRD (v1.1: 다국어 지원 포함)**

#### **1. 프로젝트 개요 (Overview) 🚀**

* **프로젝트명**: Notion-Powered Blog Automator
* **목표**: Notion을 CMS로 사용하여 한국어/영어 콘텐츠를 관리하고, GitHub Actions를 통해 10분마다 자동으로 GitHub Pages에 배포되는 **다국어** 개인 블로그를 개발한다.
* **핵심 컨셉**: 별도의 관리자 페이지 없이, 글쓰기는 Notion에서, 배포와 호스팅은 GitHub에서 자동으로 처리되는 완전 자동화 워크플로우 구축.
* **레퍼런스 사이트**: [https://hong-minji.github.io/works/](https://hong-minji.github.io/works/)
    * *모든 UI/UX 및 기능은 이 레퍼런스 사이트와 동일하게 구현하는 것을 목표로 한다.*

---

#### **2. 기술 아키텍처 및 연동 요구사항 (Architecture) 🏗️**

* **콘텐츠 소스 (CMS)**: **Notion**
    * **[변경]** 다국어 콘텐츠 관리를 위해 데이터베이스 속성을 확장한다.
    * **필수 속성**:
        * `Title (KO)` (제목, 한국어)
        * `Title (EN)` (텍스트, 영어 제목)
        * `Slug` (텍스트, URL 경로 - **언어와 무관하게 공통으로 사용**)
        * `Status` (선택: Published/Draft)
        * `Date` (날짜)
        * `Tags (KO)` (다중 선택, 한국어 태그)
        * `Tags (EN)` (다중 선택, 영어 태그)
        * `Description (KO)` (텍스트, 한국어 설명)
        * `Description (EN)` (텍스트, 영어 설명)
    * **콘텐츠 본문**: Notion 페이지 본문 자체에 한국어 내용을 작성하고, 그 아래에 구분선을 넣은 뒤 영어 내용을 이어서 작성하는 방식을 권장. (스크립트에서 이를 파싱하여 분리)

* **자동화 (CI/CD)**: **GitHub Actions**
    * **트리거**: 10분 간격의 스케줄 (`cron: '*/10 * * * *'`)
    * **작업 흐름**:
        1.  Notion API를 통해 `Status`가 'Published'인 글의 **모든 언어 데이터**를 가져온다.
        2.  가져온 데이터를 언어별(ko, en) 정적 파일(마크다운 또는 JSON)로 변환한다.
        3.  프로젝트를 빌드하여 정적 파일(HTML, CSS 등)을 생성한다.
        4.  빌드 결과물을 `gh-pages` 브랜치에 배포한다.

* **호스팅 (Hosting)**: **GitHub Pages**
    * `gh-pages` 브랜치를 소스로 하여 웹사이트를 서비스한다.

* **URL 구조**: 언어별 경로를 사용한다.
    * 한국어: `your-blog.github.io/ko/post-slug`
    * 영어: `your-blog.github.io/en/post-slug`
    * 기본 접속 시 브라우저 언어 설정에 따라 `ko` 또는 `en`으로 자동 리디렉션. (기본값은 `ko`)

#### **3. 기능 요구사항 상세 (Features) ✨**

* 프로필 영역 (좌측)
    * 프로필 이미지: 원형 이미지.
    * 이름: "" (언어 변경과 상관없이 고정 또는 언어별 설정 가능)
    * 자기소개: **언어 설정에 따라 한국어/영어로 표시**.
    * Copyright: "© Minseung Kwon All rights reserved." (공통)

##### **3.2. 포스트 리스트 및 페이지네이션**

* 포스트 표시: 한 페이지에 최신 글 순서로 **5개**씩 표시.
* 페이지 이동 버튼:
    * "<- PREV" / "-> NEXT" (영어)
    * "<- 이전" / "다음 ->" (한국어) - **언어 설정에 따라 텍스트 변경**

##### **3.3. 다크/라이트 모드 전환**

* (기존 요구사항과 동일)

##### **3.4. 포스트 상세 페이지 및 콘텐츠 렌더링**

* 콘텐츠 렌더링: Notion에서 작성한 콘텐츠가 올바르게 렌더링.
* 콘텐츠 하단 정보: 발행일과 태그가 **현재 선택된 언어에 맞게** 표시되어야 함.

##### **3.5. 푸터 (Footer)**

* "© 2026 Minseung Kwon. Designed by Minseung Kwon. Licensed under MIT."

##### **3.6. 다국어 (i18n) 기능 🌐**

* **지원 언어**: **한국어 (ko)**, **영어 (en)**
* **언어 전환 UI**:
    * **위치**: 프로필 영역의 다크/라이트 모드 아이콘 옆
    * **형태**: "KO / EN" 형태의 텍스트 버튼 또는 토글 스위치
    * **기능**: 클릭 시 사이트 전체 언어가 즉시 전환되며, URL 경로도 변경되어야 함 (`/ko` ↔ `/en`). 사용자의 언어 선택은 브라우저에 저장되어 재방문 시에도 유지.
* **번역 대상**:
    * **동적 콘텐츠 (Notion)**: 글 제목, 본문, 태그, 설명 등 Notion DB에서 가져오는 모든 텍스트.
    * **정적 UI 텍스트 (Chrome)**: `PREV/NEXT` 버튼, 프로필 자기소개, Footer 등 코드에 하드코딩된 모든 텍스트. 이는 `ko.json`, `en.json` 같은 언어 파일을 통해 관리한다.
